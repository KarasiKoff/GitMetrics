## 2. Инструкция по быстрому запуску (развёртывание «из нуля»)

> Проект — одноконтейнерная CLI-утилита. Используется `docker build` + `docker run` (без docker-compose).

### Вариант A: Docker (рекомендуется для проверки)

```bash
# 1. Клонировать репозиторий
git clone https://github.com/KarasiKoff/GitMetrics.git
cd GitMetrics

# 2. Создать файл конфигурации
cp .env.example .env          # Linux / macOS
# copy .env.example .env      # Windows
```

**Заполнить `.env`:**

```env
GITHUB_TOKEN=ghp_ваш_токен_github
LOG_LEVEL=INFO
LOG_DIR=logs
```

Токен: GitHub → Settings → Developer settings → Personal access tokens (scope: `repo` для приватных репозиториев или `public_repo` для публичных).

```bash
# 3. Собрать Docker-образ
docker build -t gitmetrics .

# 4. Проверить справку
docker run --rm gitmetrics --help

# 5. Запустить аудит
docker run --rm --env-file .env gitmetrics audit \
  --owner KarasiKoff \
  --repo GitMetrics \
  --since 2026-01-01 \
  --until 2026-06-25 \
  --format json,md
```

**Ожидаемый результат:** сводка в stdout, файлы `report.json` и `report.md` в рабочей директории контейнера.

Сохранить отчёты и логи на хост:

```bash
docker run --rm --env-file .env \
  -v "$(pwd)/reports:/app/reports" \
  -v "$(pwd)/logs:/app/logs" \
  gitmetrics audit \
  --owner KarasiKoff \
  --repo GitMetrics \
  --since 2026-01-01 \
  --until 2026-06-25 \
  --output /app/reports
```

Windows (PowerShell):

```powershell
docker run --rm --env-file .env -v "${PWD}/reports:/app/reports" -v "${PWD}/logs:/app/logs" gitmetrics audit --owner KarasiKoff --repo GitMetrics --since 2026-01-01 --until 2026-06-25 --output /app/reports
```

### Проверка обработки ошибок (неверный токен)

```bash
# В .env временно: GITHUB_TOKEN=invalid_token
docker run --rm --env-file .env gitmetrics audit \
  --owner KarasiKoff --repo GitMetrics \
  --since 2026-01-01 --until 2026-06-25
```

**Ожидаемое поведение:**
- Сообщение пользователю: «Ошибка: неверный GitHub-токен…»
- Exit code: `1`
- Запись **ERROR** в stderr и в `logs/gitmetrics.log`
- Приложение не падает с необработанным traceback
