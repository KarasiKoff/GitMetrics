## 1. Текущий статус разработки ядра

Ядро продукта реализовано и работает end-to-end. Ключевые функции бизнес-логики:

- **Загрузка коммитов** через GitHub REST API (`GET /repos/{owner}/{repo}/commits`) с пагинацией по заголовку `Link`, фильтрацией по периоду `since`/`until`.
- **Валидация Conventional Commits** — проверка сообщений по regex (`feat:`, `fix:`, `docs:` и др.), подсчёт доли валидных коммитов.
- **Агрегация метрик** — общее число коммитов, разбивка по типам, статистика по авторам (количество коммитов, % conventional).
- **Генерация отчётов** — `report.json` (полная структура `AuditReport`) и `report.md` (сводка, таблица авторов, топ нарушений).
- **Оркестрация** — use case `AuditUseCase`: fetch → analyze → write.
- **CLI** — команда `gitmetrics audit` подключена к backend, выполняет полный цикл аудита (merged PR #12).

**Проверка:** аудит репозитория `KarasiKoff/GitMetrics` за период 2026-01-01 — 2026-06-25: 30 коммитов, 16 conventional (53.33%), отчёты `report.json` и `report.md` созданы успешно.

**Тестовое покрытие:** 22 теста (unit + integration), `pytest` и `ruff check` проходят без ошибок.

## 2. Интеграция стандартов обслуживания

### Изоляция секретов

Секреты вынесены в переменные окружения. Токен GitHub загружается из `.env` через `python-dotenv`. В репозитории хранится только шаблон `.env.example`; файл `.env` добавлен в `.gitignore`.

**Листинг `.env.example`:**

```env
GITHUB_TOKEN=ghp_your_token_here
LOG_LEVEL=INFO
LOG_DIR=logs
```

Загрузка в коде (`src/gitmetrics/infrastructure/config.py`):

- `load_dotenv()` читает `.env` при старте приложения.
- `Settings.github_token` — обязательное поле; при отсутствии токена выбрасывается `ValueError` с понятным сообщением.
- `Settings.log_level` — уровень логирования (по умолчанию `INFO`).
- `Settings.log_dir` — директория для log-файла (по умолчанию `logs`).

В исходном коде **отсутствуют захардкоженные токены и пароли**.

Сгенерированные артефакты (`logs/`, `report.json`, `report.md`) также добавлены в `.gitignore` и не попадают в репозиторий.

---

### Логирование

Инициализация выполняется в модуле `src/gitmetrics/infrastructure/logging.py` функцией `setup_logging(level: str, log_dir: str)`:

1. Уровень логирования берётся из `LOG_LEVEL` в `.env` (через `Settings`).
2. Директория для файла — из `LOG_DIR` (по умолчанию `logs/`).
3. Настраиваются два обработчика с единым форматом:
   ```
   %(asctime)s | %(levelname)s | %(name)s | %(message)s
   ```
   - **stderr** — вывод в консоль / `docker logs`;
   - **файл** `{LOG_DIR}/gitmetrics.log` — дублирование логов на диск.

**Разделение по уровням:**

| Уровень | Сценарий |
|---|---|
| **INFO** | Старт/конец аудита, количество загруженных коммитов |
| **WARNING** | Пустой результат запроса, превышение rate limit GitHub API |
| **ERROR** | Ошибки аутентификации (401), репозиторий не найден (404), сбои API, ошибки записи отчёта |

Вызов `setup_logging(settings.log_level, settings.log_dir)` производится в команде `audit` (`cli.py`) до начала бизнес-логики. Дополнительное логирование реализовано в `GitHubClient` при обращении к API.

### Контейнеризация

Приложение упаковано в Docker-образ с использованием multi-stage build (builder + runtime slim).

**Dockerfile:**

```dockerfile
# syntax=docker/dockerfile:1

# --- Stage 1: builder — установка зависимостей и пакета ---
FROM python:3.12-slim AS builder

WORKDIR /app

COPY pyproject.toml README.md ./
COPY src/ src/

RUN pip install --no-cache-dir --upgrade pip \
    && pip install --no-cache-dir .

# --- Stage 2: runtime — минимальный образ только для запуска CLI ---
FROM python:3.12-slim AS runtime

ENV PYTHONDONTWRITEBYTECODE=1 \
    PYTHONUNBUFFERED=1 \
    LOG_DIR=/app/logs

WORKDIR /app

RUN mkdir -p /app/logs

COPY --from=builder /usr/local/lib/python3.12/site-packages /usr/local/lib/python3.12/site-packages
COPY --from=builder /usr/local/bin/gitmetrics /usr/local/bin/gitmetrics

ENTRYPOINT ["gitmetrics"]
CMD ["--help"]
```

**Пример запуска:**

```bash
docker build -t gitmetrics .

docker run --rm --env-file .env gitmetrics audit \
  --owner KarasiKoff \
  --repo GitMetrics \
  --since 2026-01-01 \
  --until 2026-06-25
```

Сохранение логов с хоста:

```bash
docker run --rm --env-file .env -v "%cd%/logs:/app/logs" gitmetrics audit \
  --owner KarasiKoff \
  --repo GitMetrics \
  --since 2026-01-01 \
  --until 2026-06-25
```

Секреты передаются через `--env-file .env` и не включаются в Docker-образ.
